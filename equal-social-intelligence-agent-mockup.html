Build spec · for the team
Account metrics — daily GET /{ig-user-id}/insights with period=day. Reach, follower_count, views. Meta retains only 90 days — snapshot to keep history.
Post metrics — lifetime GET /{media-id}/insights returns cumulative totals, not a daily series. Daily per-post velocity = today's snapshot − yesterday's.
Performance index send rate ÷ trailing-10 median (same format) × 100. 100 = on par. Median, not mean, so one viral post can't move the bar.
Maturity window Judge a post at T+72h, never at publish. Reels keep accruing reach for days — scoring early creates false slumps.
Jobs Snapshot daily 02:00 IST. Digest weekly Mon 08:00 IST → pipeline computes flags → Claude writes the read.
Alert thresholds Slump: 2 matured posts < 70% of median. Cliff: 7-day reach < −25%. Breakout: any post > 2× median.
Division of labour: the pipeline does the maths (snapshots, deltas, indices, thresholds) — deterministic and auditable. Claude does the narration (why winners won, what the stalls share, the one change to make). The model never computes a number it could get wrong.
This is a static design mockup with sample data — a visual target for the build, not a live dashboard.
